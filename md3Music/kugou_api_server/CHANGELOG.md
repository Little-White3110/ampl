# KuGou API Server 更新日志

## v1.5.2 (2026-06-29)

### 🔒 安全修复

- **禁用 `/audio/proxy` 音频代理端点** - 该端点导致服务器流量耗尽（出站流量 14.6 GB），因为所有音频文件都经过服务器代理转发。现在客户端应直接使用 `/song/url` 返回的 CDN URL 播放，不再经过服务器
- **添加请求频率限制** - 防止 API 被滥用。每个 IP 每 15 分钟最多 100 个请求，超过限制返回 429 错误
- **移除调试日志** - 移除 `/song/url` 接口的完整响应日志输出，减少日志文件大小

### 📊 预期效果

| 指标 | 修改前 | 修改后 |
|---|---|---|
| 出站流量 | 14.6 GB | < 100 MB |
| 服务器负载 | 高（代理音频） | 低（只返回 JSON） |
| 响应速度 | 慢（下载 + 转发） | 快（直接返回 URL） |

**部署说明**

1. 重新构建 Docker 镜像：`docker build -t kugou-api:latest .`
2. 重启容器：`docker restart kugou-api`
3. 或直接使用 pm2 重启：`pm2 restart kugou-api`

**验证步骤**

1. 测试音频代理是否已禁用：
   ```bash
   curl "http://your-server/audio/proxy?url=test"
   # 应返回 403 错误
   ```

2. 监控服务器流量：
   - Docker: `docker stats kugou-api`
   - 观察出站流量是否显著下降

3. 检查客户端播放是否正常：
   - 打开 Flutter 应用
   - 播放歌曲，确认能正常播放
   - 检查日志，确认使用的是直接 URL
