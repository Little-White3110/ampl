# MD3Music 更新日志

## v2.5.0 (2026-07-06)

### 🎉 新功能
- **嵌入式 Node.js 服务器** - App 内置完整 Node.js 服务器，无需外部服务器即可运行
- **本地 + 云端混合架构** - 所有 API 请求在本地处理，仅登录/同步走云端，月流量 < 1MB
- **多架构支持** - 同时支持 armeabi-v7a（32位）、arm64-v8a（64位）、x86_64（模拟器）

### 🔧 技术改进
- 使用 `nodejs-mobile` 框架在 Android 中运行嵌入式 Node.js
- 采用 Toyo 分步初始化方案（InitializeNodeWithArgs → CreateIsolate → LoadEnvironment → evalScript）
- 服务器代码通过 esbuild 打包为 `server_bundle.js`，作为 Flutter asset 打入 APK
- 优化 `.gitignore`，避免提交构建缓存和临时文件

- 重写 README.md，详细说明新架构和构建流程
- 添加项目结构说明和常见问题解答

## v2.2.1 (2026-06-25)

### 🔧 修复
- **修复多租户数据隔离严重漏洞** - 用户登录后可能看到其他用户的数据（头像、昵称、歌单、收藏等）。原因：`SharedPreferences` 使用全局键名存储认证信息，改为用户隔离的键名（`kugou_token_{userid}`），彻底解决多账号数据混淆问题
- **修复「添加到歌单」不显示歌曲数量** - 对话框中歌单始终显示"0 首"，原因是 UI 直接读取 API 原始 JSON 字段名不匹配，改为使用 `KugouPlaylistBrief` 模型正确解析 `songcount` 字段
- **移除评论区点赞数量** - 酷狗 API 不返回点赞数（始终为 0），移除无效的点赞图标及数字，评论区更简洁

### ⚡ 优化
- **退出登录时清除所有用户数据** - `logout()` 现在清除内存中的用户信息、VIP 信息、历史记录、头像缓存等，避免数据残留
- **设置页新增「数据迁移」入口** - 用户可一键清除旧版本污染的 `SharedPreferences` 数据，修复数据混乱后无需手动清除应用数据

## v2.2.0 (2026-06-17)

### 🎉 新功能
- **播放器「添加到歌单」** - 播放器页面右上角三点菜单新增「添加到歌单」入口，点击后弹出用户创建的歌单列表，可将当前播放歌曲添加至所选歌单

### 🔧 修复
- **修复荣耀/vivo 手机播放时出现通知提示音** - 优化通知渠道配置，设置静音并禁用震动，兼容各品牌机型
- **修复荣耀平板 V8 Pro 播放时音量忽高忽低** - 移除音频焦点 duck 事件的音量调整逻辑，避免频繁 duck/unduck 导致音量波动
- **修复歌单列表显示不全** - 「添加到歌单」弹窗只显示用户自己创建的歌单，过滤掉收藏的歌单和专辑
- **修复添加到歌单失败** - 修正 API 调用参数格式（歌名|hash|albumId|albumAudioId）
- **修复「我创建的歌单」无批量删除按钮** - 歌单分区头部新增「管理」按钮，支持批量选择并删除歌单

### ⚡ 优化
- **优化添加到歌单/收藏的同步速度** - 改为乐观更新 + 后台同步，点击后瞬间响应，不再阻塞等待服务器返回
- 搜索功能添加缓存机制，优化页面切换体验
- 发现页添加数据新鲜度缓存，减少重复请求
